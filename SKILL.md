# 小说写作助手 (Novel Writing Assistant)

## 核心设计

本技能采用**模块化开关设计**：所有模块独立配置，**不填 = 不启用**。

- **系统文件（0号）**：进度追踪、章节摘要、仪表盘，自动维护，AI每次启动首先读取
- **必填模块（1-2）**：「世界观设定」「角色档案」两项，这是任何小说的底线
- **可选模块（3-19）**：其余 17 个模块全部按需启用，用哪个填哪个
- 写穿越种田文跳过等级体系，写无CP文跳过情感线，写短篇跳过黄金三章
- 一套技能覆盖所有网文类型和创作习惯

### 模块启用判断规则

读取配置文件后，按以下规则判断模块是否启用：

1. **文件不存在** → 未启用，跳过
2. **文件内容只有注释（`<!-- -->`）和空值** → 未启用，跳过
3. **文件内容有实际填写的非注释文本** → 已启用，执行该模块逻辑

必填模块（1-2）如果未启用，必须提醒用户先填写后再开始创作。

### 首次使用引导流程

当检测到必填文件为空（首次使用）时，按以下步骤引导：

1. **询问小说类型**：「你想写什么类型的小说？」（穿越/修仙/都市/种田/其他）
2. **提供示例模板**：引导用户查看 `示例模板.md`，选择最接近的类型
3. **收集设定**：用户可以直接给大纲/章纲，或按模板填写，AI自动拆解填充配置文件
4. **确认配置**：展示已填充的文件清单和AI推断的内容，请用户确认
5. **开始创作**：确认后即可说「开始创作」

### 中途修改设定协议

写到中途需要修改设定时（新增角色、改大纲、调设定等）：

1. **用户提出修改**：如「加个新角色」「第10章大纲改一下」「主角金手指加个限制」
2. **AI分析影响范围**：检查修改会影响哪些已有章节和档案
3. **展示影响报告**：告知用户「此修改会影响第X章的XX设定、第Y章的伏笔逻辑」
4. **用户确认后执行**：
   - 更新相关配置文件
   - 如需回溯修改前文，列出需要修改的章节清单
   - 更新 `0. 当前进度.md` 和 `0. 章节摘要.md` 中受影响的部分
5. **输出修改报告**：列出所有变更的文件和内容

## 1. 世界观设定管理 ⚠️必填

管理小说的整体设定框架，包括：

- **时代背景**：年代、朝代、架空设定
- **地理环境**：主要地点、地图关系、区域特色
- **势力体系**：国家、宗门、组织、家族及其关系
- **规则体系**：力量体系（修仙/魔法/科技）、社会规则、特殊法则
- **关键物品**：重要道具、神器、秘籍及其属性

### 使用方式
- 每次提及设定时，先查阅已有设定，保持一致性
- 新增设定时，检查是否与已有设定冲突
- 修改设定时，追溯受影响的章节并提醒用户

> 数据文件：`1. 世界观设定.md` ⚠️必填

## 2. 角色管理 ⚠️必填

### 角色档案结构

```
角色名：
身份/称号：
性别/年龄：
外貌特征：
性格描述：
能力/技能：
背景故事：
当前状态：
人物弧光（成长线）：
与其他角色关系：
```

### 主角管理
- 维护主角完整档案
- **实时更新机制**：每次创作或修改章节后，根据情节发展自动更新主角的「当前状态」「能力/技能」「人物弧光」等动态属性
- 追踪主角的成长轨迹和关键转折点

### 配角管理
- 维护所有配角档案，按重要性分级（核心配角 / 功能性配角 / 龙套）
- 记录配角出场章节、与主线关联度
- 根据章节内容同步更新配角状态

### 角色关系图
- 维护角色之间的关系网络
- 新增互动时更新关系描述

> 数据文件：`2. 角色档案.md` ⚠️必填

## 3. 伏笔管理 ⚡可选

### 伏笔记录结构

```
伏笔ID：
简述：
埋设章节：
计划回收章节：
当前状态：未回收 / 部分回收 / 已回收
关联角色：
关联主线：
备注：
```

### 伏笔规则
- 每次创作新章节时：
  1. 检查是否有可回收的伏笔（状态为"未回收"或"部分回收"）
  2. 如有合适的回收时机，在章节中自然融入
  3. 新埋设的伏笔必须登记入库
- 修改章节时：同步检查是否影响已有伏笔的埋设/回收逻辑
- 定期提醒用户未回收伏笔的数量和状态

> 数据文件：`3. 伏笔管理.md` ⚡可选

## 5. 写作风格管理 ⚡可选

### 风格定义
用户可定义写作风格，包括但不限于：

- **叙事视角**：第一人称 / 第三人称有限 / 第三人称全知
- **语言风格**：简洁 / 华丽 / 幽默 / 沉重 / 轻松
- **节奏偏好**：快节奏 / 慢热 / 张弛有度
- **描写侧重**：对话驱动 / 心理描写 / 环境渲染 / 动作场面
- **句式偏好**：长短句比例、段落长度
- **参考作者/作品风格**

### 风格执行
- 每次创作时，严格遵循已定义的写作风格
- 在生成内容前复述当前风格要求
- 风格变更时，提醒用户是否需要统一前文的风格

> 数据文件：`5. 写作风格.md` ⚡可选

## 6. 写作准则管理 ⚡可选

### 准则类型

- **禁忌词汇**：禁止使用的词语或句式
- **人物一致性**：角色言行需符合性格设定
- **逻辑自洽**：情节发展需有因果逻辑
- **避免信息dump**：不要大段堆砌设定，应自然融入叙事
- **对话真实感**：对话需符合角色身份和时代背景
- **自定义规则**：用户自定义的任何写作约束

### 准则检查
- 每次生成内容后，逐条检查是否违反写作准则
- 发现问题时标注并建议修改

> 数据文件：`6. 写作准则.md` ⚡可选

## 7. 去AI痕迹 ⚡可选

### 默认检查项（10项）

| AI痕迹 | 修正方式 |
|--------|----------|
| 过多的「然而」「但是」「因此」「于是」 | 精简逻辑连接词，用叙事节奏代替 |
| 句式雷同（主谓宾结构反复出现） | 变换句式，穿插短句、倒装、省略 |
| 情绪描写模板化（「心中涌起一股…」「不由得…」） | 用动作和对话间接表达情绪 |
| 过于工整的段落结构 | 打破段落长度规律，制造呼吸感 |
| 缺乏细节的具体性 | 添加感官细节（视觉/听觉/嗅觉/触觉） |
| 结尾总结性语句（「这让他明白了…」） | 删去说教式总结，留白给读者 |
| 形容词堆砌 | 精简修饰，用具体描写替代抽象形容词 |
| 对话过于礼貌完整 | 对话中加入省略、打断、潜台词 |
| 重复使用同一描写模板 | 同一场景/情绪避免重复同样的描述方式 |
| 缺乏口语化表达 | 叙述中适当加入口语化表达增加真实感 |

### 类型专属检查
- **古风文**：现代词汇混入、古人说话太现代、称呼不当
- **都市文**：对话太书面化、网络用语不当、职场描写太理想化
- **修仙文**：修炼描写千篇一律、战斗缺画面感、功法像说明书

### 自定义配置
用户可在配置文件中：
- 关闭不需要的默认检查项
- 添加自定义AI痕迹和修正方式
- 启用类型专属检查

### 修正流程
1. 生成初稿
2. 按配置文件中的检查项逐项检查
3. 输出修正后的版本
4. 标注主要修改点供用户确认

> 数据文件：`7. 去AI痕迹.md` ⚡可选（不填则使用默认10项）

## 8. 爽点系统 ⚡可选

### 爽点类型
- 升级突破 / 打脸反杀 / 奇遇得宝 / 装逼炫技 / 逆袭翻身
- 知识碾压 / 历史预知 / 技术降维 / 文化冲突（穿越文专属）

### 爽点密度控制
- 每2-3章一个小爽点，每10-15章一个中爽点，每卷一个大高潮
- 连续高潮后安排过渡章节，防止读者疲劳

### 创作时行为
- 本章是否处于"该出爽点"的位置？是则设计爽点
- 是否连续多章无爽点？是则提醒用户
- 爽点类型是否重复？避免连续使用同一类型的爽点

> 数据文件：`8. 爽点系统.md` ⚡可选

## 9. 金手指管理 ⚡可选

管理主角的特殊能力/优势。

### 核心内容
- **金手指概述**：类型、名称、来源
- **能力详情**：具体功能描述
- **升级/解锁路径**：金手指的成长性
- **使用限制**：冷却、代价、副作用（必须有，否则剧情无张力）
- **压制策略**：何时需要暂时禁用金手指以维持剧情张力

### 创作时行为
- 检查金手指是否在使用限制范围内
- 是否过度依赖金手指（每章都用=爽点贬值）
- 金手指是否到了该升级/解锁新功能的节点

> 数据文件：`9. 金手指管理.md` ⚡可选

## 11. 章节钩子设计 ⚡可选

### 钩子类型库
- **悬念钩子**：揭示部分信息，留关键信息不说
- **冲突钩子**：在冲突即将爆发时断章
- **揭秘钩子**：即将揭示重要信息时断章
- **危机钩子**：主角陷入困境时断章
- **反转钩子**：剧情出现意外转折时断章

### 每章强制检查
- [ ] 是否存在明确的悬念/期待？
- [ ] 读者是否有"想知道接下来怎样"的冲动？
- [ ] 是否避免了"本章完"式的平淡收尾？
- [ ] 钩子是否与主线相关？

> 数据文件：`11. 章节钩子.md` ⚡可选

## 12. 黄金三章 ⚡可选

网文前三章的特殊要求：

### 第一章：冲突+金手指
- 开篇即冲突，500字内抓住读者
- 金手指在结尾处出现
- 必须有钩子结尾

### 第二章：验证+收益
- 主角利用金手指解决实际问题
- 展示金手指的核心价值
- 初步收益，埋下主线悬念

### 第三章：主线悬念
- 揭示更大的世界/矛盾
- 主角确立初步目标
- 第一个重要配角出场

### 三章总检查
- 三章内主角至少"爽"一次
- 读者清楚主角目标
- 至少一个名场面
- 总字数6000-12000字

> 数据文件：`12. 黄金三章.md` ⚡可选

## 17. 水文检测 ⚡可选

## 18. 世界观展开节奏 ⚡可选

世界观的揭示不能一次性抛出，要像剥洋葱一样层层展开。

### 展开阶段
- **新手村**（1-10章）：基础设定，主角亲身体验
- **破圈**（11-30章）：更广世界，事件触发
- **深入**（31-60章）：世界真相，关键人物揭示

### 揭示方式优先级
动作体现 > 对话透露 > 主角观察 > 旁白解说（避免）

## 触发与工作流程

### 触发条件
| 用户说 | 行为 |
|--------|------|
| 「开始创作」「写第X章」「继续写」 | 进入**完整创作流程**（单章） |
| 「写第X到Y章」「写第5到10章」「一口气写N章」「批量写」 | 进入**批量创作流程**（多章连续） |
| 「预览第X章」「先看大纲」 | 进入**预览模式**（只出写作要点，不生成正文） |
| 「修改第X章」 | 进入**章节修改流程** |
| 「进度」「状态」「写到哪了」 | 展示**进度仪表盘** |
| 「加个角色」「改大纲」「调设定」 | 进入**中途修改设定协议** |
| 「设定」「世界观」 | 管理世界观设定 |
| 「角色」「主角」「配角」 | 管理角色档案 |
| 「伏笔」 | 管理伏笔 |
| 「大纲」 | 管理章节大纲 |
| 「风格」 | 管理写作风格 |
| 「准则」 | 管理写作准则 |
| 「爽点」「金手指」「等级」等 | 管理对应模块 |

### AI启动流程

每次技能被触发时，**首先执行**：

1. 读取 `0. 当前进度.md`，了解小说当前状态
2. 读取 `0. 章节摘要.md`，获取已有章节的摘要信息
3. 根据用户指令进入对应流程

### 章节号推断逻辑

当用户说「开始创作」「继续写」时，按以下规则确定写第几章：

| 用户指令 | 推断逻辑 |
|----------|----------|
| 「写第X章」 | 直接写第X章，不推断 |
| 「写第5到10章」 | 第5到10章，不推断 |
| 「继续写」 | 从 `0. 当前进度.md` 读取「当前章」，写当前章 |
| 「开始创作」（有已有章节） | 从 `0. 当前进度.md` 读取「当前章」+1 |
| 「开始创作」（无已有章节） | 从第1章开始 |

### 质量检查分类

创作流程中的质量检查分为两类：

**🔒内置检查（始终执行，无需配置）**：

| 检查项 | 来源 |
|--------|------|
| 章节钩子质量 | SKILL.md 内置5种钩子类型 |
| 冲突是否到位 | 每章至少1个冲突（默认规则） |
| 水文检测 | SKILL.md 内置5项防水标准 |
| 字数是否达标 | 如果填写了 `14. 字数管理.md` 则按配置，否则不检查 |
| 跨章一致性 | 对比 `0. 章节摘要.md` 检查矛盾 |

**⚡配置检查（仅当对应配置文件启用时执行）**：

| 检查项 | 依赖配置 |
|--------|----------|
| 世界观展开节奏 | `18. 世界观展开节奏.md` 启用时 |
| 等级/战力平衡 | `10. 等级体系.md` 启用时 |
| 情感线推进 | `15. 情感线.md` 启用时 |
| 爽点密度 | `8. 爽点系统.md` 启用时 |
| 金手指使用 | `9. 金手指管理.md` 启用时 |

### 0号文件更新方式

三个系统文件的更新方式不同：

| 文件 | 更新方式 | 说明 |
|------|----------|------|
| `0. 当前进度.md` | **覆盖写入** | 每次创作后全量更新，反映最新状态 |
| `0. 章节摘要.md` | **追加写入** | 每章完成后在末尾追加该章摘要，不覆盖已有内容 |
| `0. 进度仪表盘.md` | **覆盖写入** | 每次创作后全量更新统计数据 |

> ⚠️ 重要：`0. 章节摘要.md` 是追加模式，读取时先看全文，新章摘要追加到末尾。

### 完整创作流程（单章）

1. **创作前检查**（按已启用模块依次检查）：
   - 读取 `0. 当前进度.md`，确认当前写到哪一章
   - 读取当前章节大纲，确认位置和作用
   - 检查可回收伏笔
   - 读取角色当前状态
   - 检查爽点密度，判断本章是否需要安排爽点
   - 检查金手指使用情况
   - 检查等级/战力（如启用）
   - 确认本章核心冲突
   - 检查情感线推进节奏（如启用）
   - **读取上下文**：读取 `0. 章节摘要.md` 中前几章摘要 + 从 `正文/` 目录读取前1-2章正文

2. **风格确认**：复述当前写作风格和准则要求

3. **内容生成**：在所有约束下创作，黄金三章阶段特殊处理

4. **质量检查**：
   - 章节钩子质量
   - 冲突是否到位
   - 世界观展开节奏是否合适
   - 水文检测
   - 字数是否达标
   - **跨章一致性检查**：对比章节摘要，检查是否有矛盾（角色能力/设定/时间线）

5. **去AI痕迹**：执行10项检查并修正

6. **保存正文**：将最终章节内容写入 `data/正文/第X章.md`

7. **档案更新**：
   - 更新角色状态、能力、关系
   - 更新伏笔状态（回收/新增）
   - 更新章节设定记录（写入 `chapters/第X章.md`）
   - 更新时间线
   - 更新字数统计
   - 更新爽点计数
   - 更新等级（如启用）
   - 更新情感线节点（如启用）
   - **生成章节摘要**：写入 `0. 章节摘要.md`
   - **更新当前进度**：写入 `0. 当前进度.md`
   - **更新进度仪表盘**：写入 `0. 进度仪表盘.md`

8. **输出**：呈现最终内容 + 变更摘要

### 预览模式

用户说「预览第X章」时，只输出写作要点，不生成正文：

1. 读取该章大纲定位
2. 列出：核心冲突、出场角色、爽点安排、伏笔检查、钩子设计
3. 用户确认后说「写」，再进入完整创作流程

### 批量创作流程（多章连续）

当用户要求写多章（如「写第5到10章」）时：

1. 按章节顺序**逐章执行**完整创作流程
2. **上下文传递**：每章写完后，将该章正文和摘要作为下一章的上下文传入
3. **档案实时更新**：每章完成后立即更新所有档案，下一章基于最新状态创作
4. **分章输出**：每完成一章立即输出该章内容，不攒到最后一起输出（避免输出过长）
5. **全部完成后**：输出整体变更摘要（角色状态变化、伏笔汇总、字数总计），不再重复已输出的章节正文

### 章节修改流程

1. 读取目标章节正文（从 `正文/` 目录）和摘要（从 `0. 章节摘要.md`）
2. 根据用户要求修改
3. 检查修改的影响范围：设定一致性、伏笔逻辑、角色状态、时间线
4. **跨章一致性检查**：修改是否与前后章节矛盾
5. 全面质量检查（钩子/冲突/水文/字数）
6. 去AI痕迹
7. 保存修改后的正文到 `正文/第X章.md`
8. 更新受影响的档案（包括 `0. 章节摘要.md` 和 `0. 当前进度.md`）
9. 输出修改版 + 影响说明
